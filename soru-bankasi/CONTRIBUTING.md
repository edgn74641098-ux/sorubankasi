# Katki Rehberi

Bu projeye katkida bulunmak istediginiz icin tesekkurler.

## Gelistirme Akisi

1. `main` branch'inden yeni bir branch acin.
2. Degisikliklerinizi kucuk ve anlamsal commit'lerle yapin.
3. Testleri calistirin:

```bash
php artisan test
```

4. Pull Request acin ve degisikligin amacini net yazin.

## Kod Standardi

- Mevcut proje yapisina ve isimlendirme stiline sadik kalin.
- Gereksiz buyuk refactor'lardan kacinin.
- UI degisikliklerinde mevcut tema dilini koruyun.

## PR Kontrol Listesi

- [ ] Ozellik veya bugfix net aciklandi
- [ ] Testler gecti
- [ ] Yeni migration varsa belirtildi
- [ ] Geriye donuk uyumluluk etkisi not edildi
